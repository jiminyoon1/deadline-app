# Deadline Flow (구 하루집중)

마감이 코앞에 와야 움직여지는 사람들을 위한 타임박싱 데스크탑 앱입니다. 할 일마다 마감 시간을 정해두고, 화면 위에 항상 떠 있는 타이머 위젯으로 카운트다운을 보면서 집중합니다. 클라우드 계정이나 로그인 없이 내 컴퓨터에만 기록이 저장됩니다.

**이런 분께 맞습니다**
- 마감이 있어야 일이 손에 잡히는 사람
- 뽀모도로처럼 딱딱하게 끊기지 않고, 태스크별로 유동적인 시간 배분을 원하는 사람
- 작업 중 다른 창에 가려도 타이머가 계속 눈에 보이길 원하는 사람

**주요 기능**
- 태스크 · 마감 설정: 오늘 할 일과 각각의 예상 소요 시간을 등록
- 몰입 타이머: 화면 위에 항상 떠 있는(always-on-top) 플로팅 위젯으로 카운트다운, 초과 시간도 표시
- 카페 홈: 진행 중인 태스크와 오늘의 흐름을 한눈에 보여주는 메인 화면
- 휴식 관리: 태스크 사이 휴식 타이머, 휴식이 끝나면 다음 태스크를 직접 이어서 시작
- 통계 · 패턴 분석: 날짜별 기록과 몰입이 잘 되는 시간대(prime time) 히트맵

Electron + React(JSX)로 만들어졌고, macOS(.dmg) / Windows(.exe) 데스크톱 앱으로 배포됩니다.

## 프로젝트 구조

- `src/main` — Electron 메인 프로세스. 두 창(메인 창 / 플로팅 위젯)을 생성하고, 하루 세션 상태(`sessionStore`)와 IPC 핸들러, 로컬 JSON 데이터 계층을 갖고 있습니다.
- `src/preload` — `contextBridge`로 렌더러에 `window.api`를 노출합니다.
- `src/renderer` — React UI. `index.html` → 메인 창(`App.jsx`, 홈/태스크 설정/기록·통계/설정 탭), `widget.html` → 플로팅 위젯(`WidgetApp.jsx`, 타이머/휴식/하루 요약 화면).

데이터는 `userData/records/YYYY-MM-DD.json`(날짜별 기록)과 `userData/settings.json`(X 버튼 모드, 플로우모도로 on/off)에 저장됩니다.

## 개발 모드 실행

```bash
npm install
npm run dev
```

실행하면 메인 창이 열립니다. 태스크 설정 탭에서 할 일을 등록하고 [시작]을 누르면 플로팅 위젯(always-on-top)이 뜨면서 카운트다운이 시작됩니다.

- 위젯의 X 버튼 동작(숨기기/종료)과 플로우모도로 on/off는 메인 창의 **설정** 탭에서 바꿀 수 있습니다.
- 숨기기 모드에서 위젯을 닫으면 메뉴바 트레이 아이콘(우클릭 시 "위젯 열기")으로 다시 불러올 수 있습니다.

## 빌드

```bash
# macOS (.dmg)
npm run build:mac

# Windows (.exe, NSIS 설치파일)
npm run build:win
```

빌드 산출물은 `dist/`에 생성됩니다. 1차 버전은 코드 서명·공증을 하지 않습니다(PRD 2장 기준). macOS는 Intel+Apple Silicon 겸용 universal dmg, Windows는 x64+arm64 겸용 exe 하나로 생성되므로, 지인들에게 기기별로 각각 하나씩 전달하면 됩니다.

서명이 없는 앱이라 처음 실행할 때 운영체제가 경고를 띄웁니다. 아래 방법으로 우회해서 실행하세요.

- **macOS**: 더블클릭 시 "확인되지 않은 개발자" 경고가 뜨면, 앱 아이콘을 **우클릭 → 열기**를 누르고 뜨는 대화상자에서 다시 **열기**를 선택합니다. (더블클릭만으로는 계속 차단됩니다.)
- **Windows**: 설치 파일 실행 시 SmartScreen이 "Windows에서 PC를 보호했습니다"라고 뜨면, **추가 정보**를 클릭한 뒤 **실행**을 누릅니다.

## 구현 범위

PRD 3장 기준 5개 기능(태스크/마감 설정, 몰입 타이머, 카페 홈, 휴식 관리, 통계·패턴 분석)을 구현했습니다. 백색소음 음원, 캐릭터, 클라우드 동기화 등 PRD 7장 백로그 항목은 포함되어 있지 않습니다.

PRD 8장 오픈 이슈 중 구현에 필요했던 항목은 사용자 확인을 거쳐 다음과 같이 반영했습니다.

| 항목 | 결정 |
|---|---|
| X 버튼 기본값 | 숨기기 모드 |
| 백색소음 자리 클릭 시 | "준비 중" 안내 토스트 표시 |
| 휴식 종료 후 다음 태스크 | 사용자가 [다음 태스크 시작]을 눌러야 시작 |
| 하루 초기화 시점 | 자정 기준 자동 리셋 |
