# 기술 카탈로그 — 목적별 시각 기법

값이 필요할 때 "무난한 기본값"이 아니라 여기서 목적에 맞는 기법을 고른다.
색·radius·그림자·모션 값은 `tokens.css`가 정본 (`front-design-tokens` Skill 참조).
이 문서는 tokens.css에 없는 타이포·간격 스케일과, 목적별 적용 기법을 다룬다.

## 타이포그래피 스케일 (이 앱의 표준)

현재 UI에서 정규화한 스케일. **이 밖의 크기를 새로 만들지 않는다.**

| 역할 | 크기 | 굵기 | 용례 |
|---|---|---|---|
| display | 36px | 700 + `tabular-nums` | 타이머 중앙 숫자 전용 |
| title | 18px | 700 | 페이지 타이틀 ("오늘 할 일") |
| body | 14px | 500 | 태스크명, 버튼 텍스트 |
| body | 14px | 400 | 입력 텍스트, 일반 본문 |
| caption | 12px | 500 + `tabular-nums` | 시간 표시 ("38분 20초") |
| caption | 12px | 400 | 날짜, 라벨, placeholder |
| micro | 11px | 400~500 | 위젯 등 밀도 높은 곳의 보조 정보만 |

- 한 화면에 (크기×굵기) 조합이 5종을 넘으면 위계가 무너진 것 — 회수한다.
- 13px·15px·17px 같은 중간값 발견 시 인접 표준으로 치환.
- 600과 700을 섞지 않는다. 강조 굵기는 700 하나, 중간 굵기는 500 하나.

### 목적별 타이포 기법
- **숫자가 실시간으로 바뀌는 곳** (타이머, 남은 시간): `font-variant-numeric: tabular-nums` —
  자릿수 변해도 폭이 흔들리지 않는다. 큰 숫자는 `letter-spacing: -0.02em`로 조이면 단단해 보인다.
- **라벨 vs 값**: 크기 차이가 아니라 색으로 구분 (`--color-text-sub` 라벨 + `--color-text` 값).
- **큰 제목**: 크기를 키웠으면 `line-height`를 1.2~1.3으로 줄인다. 본문 1.5를 그대로 두면 붕 뜬다.
- **완료/비활성**: 크기를 줄이지 말고 색·opacity로 가라앉힌다 (front-design-tokens의 done 규칙).

## 간격 스케일 (4px 그리드)

`4 / 8 / 12 / 16 / 20 / 24 / 32 / 48`px만 사용. 예외적으로 2px(아이콘 미세조정) 허용.
5px·7px·9px·14px·18px 발견 시 인접값으로 치환.

### 목적별 간격 기법
- **근접성이 곧 구조다**: 레이블↔값은 4~8, 행 내부 요소는 8~12, 행↔행은 구분선+상하 12~16,
  섹션↔섹션은 24~32, 카드 패딩은 24~32. 관계가 가까울수록 간격이 좁아야 한다.
- **정렬선은 하나**: 패널 안 모든 요소의 좌측 시작선을 하나로 맞춘다. 아이콘 때문에 텍스트가
  들쭉나면 아이콘 컨테이너 폭을 고정해서 텍스트 시작선을 지킨다.
- **광학 보정**: 수치가 같아도 달라 보이면 눈이 맞다. 원형 요소 옆 텍스트, 대문자·숫자 옆
  아이콘은 1~2px 광학 조정 허용 (주석으로 이유 명시).

## 깊이·표면 기법

- 표면 언어는 한 화면에 하나: 카드는 `--shadow-card` **또는** `--color-border` 보더 중 하나.
  둘 다 얹지 않는다 (현재 앱은 그림자 방식).
- 구분선(`--color-divider`)은 콘텐츠 영역 안쪽에서 끝내지 말고 좌우 패딩과 정렬 —
  끝까지 긋거나(리스트) 텍스트 시작선부터 긋거나(인셋), 화면 안에서 한 방식만.
- 눌리는 것과 눌리지 않는 것: 클릭 가능한 표면만 hover 배경 변화를 갖는다.
  정보 표시용 영역에 hover를 넣으면 사용자를 속이는 것.

## 강조 기법 (순한 것부터)

강조가 필요하면 이 사다리의 **가장 낮은 단**부터 쓴다:
1. 색 단계 올리기 (text-sub → text)
2. 굵기 (400 → 500)
3. 포인트 컬러 텍스트 (`--color-primary`) — "진행 중/액션" 의미일 때만
4. 틴트 배경 (`--color-primary-tint`) — 선택 상태
5. 크기 — 최후의 수단. 화면당 새 크기 도입은 사실상 금지

## 원형 타이머 기법

- 링 stroke 굵기는 지름의 8~10%. 트랙은 `--color-divider`, 진행분은 `--color-primary`.
- `stroke-linecap: round`로 끝을 둥글게 — 얇은 사각 끝은 싸구려 느낌의 주범.
- 12시 방향 시작 (`transform: rotate(-90deg)`).
- 중앙 숫자와 링 사이 여백을 충분히 — 숫자가 링에 닿을 듯하면 답답해 보인다.
