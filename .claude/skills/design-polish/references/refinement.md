# 고도화 프로토콜 — 기능 완성본을 다듬는 순서

전제: 레이아웃·인터랙션·상태가 전부 동작하는 버전이 있다. 이제 순서대로 다듬는다.
**순서를 지키는 이유**: 뒤 단계(디테일·모션)는 앞 단계(구조·위계)가 바뀌면 다시 해야 한다.
역순으로 하면 같은 일을 두 번 한다.

각 단계는 "실제 렌더 화면을 보고 → 고치고 → 다시 보고" 루프다. 코드만 보고 통과시키지 않는다.

## 0단계 — 전수조사 (고치지 말고 목록만)

대상 화면의 `.module.css`에서 실제 사용 값을 뽑는다:
- font-size × font-weight 조합 목록
- gap/padding/margin 값 목록
- border-radius, 그림자, hex 직접 사용 목록

목록이 곧 진단서다: 크기 조합 5종 초과, 4px 그리드 밖의 간격, 토큰 미사용 hex, 어중간한
radius가 전부 수리 대상. 이 목록 없이 눈대중으로 다듬기 시작하면 반드시 빠뜨린다.

## 1단계 — 통일 (스케일로 수렴)

- 조사된 값을 `techniques.md`의 타이포/간격 스케일과 `tokens.css` 토큰으로 치환.
- 같은 역할인데 다른 스타일인 요소(버튼 2종이 미묘하게 다름 등)를 하나로 합친다.
- 이 단계가 끝나면 화면의 "값 어휘"가 절반 이하로 줄어야 정상.

## 2단계 — 위계 (한 걸음 물러나서)

스크린샷을 처음 보는 사람처럼 보고 답한다:
- 0.5초 안에 눈이 가는 곳이 이 화면의 주인공(타이머/진행 중 태스크)인가?
- 두 번째, 세 번째로 읽히는 것이 실제 중요도 2, 3위인가?
- 아니라면: 위에 있는 걸 키우지 말고 **경쟁자를 가라앉힌다** (색 단계 내리기, 굵기 회수).
- 강조 사다리(techniques.md) 위반 — 크기로 해결한 곳을 색·굵기로 낮출 수 있는지 확인.

## 3단계 — 정렬·광학 보정

- 좌측 정렬선 하나로 수렴하는지 픽셀 단위 확인 (아이콘 컨테이너 폭 고정 기법).
- 수직 중심 정렬: 행 안의 텍스트·아이콘·버튼이 같은 중심선에 있는가.
- 광학 보정: 수치상 맞는데 어긋나 보이는 곳 1~2px 조정 (주석으로 이유 남김).
- 구분선·보더의 시작/끝 위치가 화면 안에서 한 방식인가.

## 4단계 — 상태·디테일

- 모든 인터랙티브 요소의 hover/pressed/disabled/focus-visible 확인
  (`front-design-tokens`의 상태 규칙이 기준).
- placeholder 색, 비활성 버튼, 빈 목록(empty state) 같은 엣지 상태를 실제로 띄워본다.
- 텍스트 오버플로: 긴 태스크명, 긴 시간 문자열에서 레이아웃이 깨지는가 (`text-overflow: ellipsis`).

## 5단계 — 모션 (마지막)

- 상태 변화 요소에만 `--transition-fast`. `transition: all` 금지.
- 등장·완료 같은 이벤트 애니메이션은 한 화면에 1~2개까지. 스프링은
  front-design-tokens의 커브 사용, `prefers-reduced-motion` 대응 확인.

## 종료 조건

`checklist.md` 전 항목 통과 + 사용자에게 보여줄 요약(바꾼 것, 주요 선택의 근거 2~3줄) 준비.
한 단계에서 세 번 이상 루프를 돌고 있으면 멈추고 사용자에게 방향을 확인한다 —
취향 문제일 수 있다.
