# 디자인 검증 체크리스트 — 제출 전에 던지는 질문

전부 "예"여야 제출한다. "아니오"가 나오면 해당 단계(refinement.md)로 돌아간다.
실제 렌더 화면(스크린샷)을 보면서 답한다 — 코드 리뷰로 대체하지 않는다.

## 첫인상 (3초 테스트)

- [ ] 처음 보는 사람의 눈이 0.5초 안에 화면의 주인공(타이머/진행 중 태스크)으로 가는가?
- [ ] 화면이 "생성기가 뽑은 관리자 템플릿"이 아니라 "이 앱만의 화면"으로 보이는가?
- [ ] 사용자가 준 시안/이미지가 있다면, 나란히 놓고 봤을 때 같은 화면이라고 느껴지는가?

## 타이포그래피

- [ ] (크기 × 굵기) 조합이 5종 이하인가? 전부 techniques.md 스케일 안에 있는가?
- [ ] 실시간 숫자(타이머·시간)에 tabular-nums가 적용됐는가? 자릿수가 바뀌어도 안 흔들리는가?
- [ ] 볼드(700)가 화면당 한두 곳인가? 라벨/보조 정보는 크기가 아니라 회색 단계로 구분했는가?
- [ ] 본문 텍스트와 배경의 대비가 4.5:1 이상인가? (text-faint는 placeholder·비활성에만)

## 간격·정렬

- [ ] 모든 간격이 4px 그리드(4/8/12/16/20/24/32/48) 위에 있는가?
- [ ] 관련 깊은 것끼리 가깝고 무관한 것끼리 먼가? (간격이 관계를 표현하는가, 균일한 벽인가)
- [ ] 패널 안 요소들의 좌측 시작선이 하나로 정렬되는가? 행 내부는 수직 중심이 맞는가?
- [ ] 바깥 여백 > 섹션 간격 > 그룹 간격 > 요소 간격 위계가 지켜지는가?

## 색·표면

- [ ] hex 직접 사용이 없는가? (허용 예외: 유색 배경 위 흰 획)
- [ ] `--color-primary`가 "진행 중/주요 액션" 의미로만 쓰였는가? 장식용 오렌지가 없는가?
- [ ] 한 화면의 표면 언어가 하나인가? (그림자 or 보더, 구분선 방식 통일)
- [ ] radius가 요소 크기에 비례하는가? (tokens.css의 sm/md/lg 안에서)

## 상태·인터랙션

- [ ] 클릭 가능한 모든 요소에 hover와 pressed가 있는가? 클릭 불가능한 요소에는 없는가?
- [ ] disabled·완료·선택 상태가 명확히 구분되는가? (완료 행이 미완료보다 무겁지 않은가)
- [ ] focus-visible 링이 살아 있는가? (outline: none으로 끈 곳이 없는가)
- [ ] 클릭 타겟이 32×32px 이상인가?

## 엣지 케이스

- [ ] 긴 텍스트(태스크명 40자)에서 레이아웃이 안 깨지는가?
- [ ] 빈 상태(태스크 0개, 진행 중 없음)가 디자인돼 있는가?
- [ ] 전이·애니메이션이 있고, prefers-reduced-motion에 대응하는가?

## 마지막 질문

- [ ] 지금 설명하라면, 이 화면의 주요 선택 세 가지에 각각 한 줄의 근거를 댈 수 있는가?
- [ ] 뭔가 하나를 **빼도** 화면이 성립하는가? 성립하면 빼라. (절제 테스트)
