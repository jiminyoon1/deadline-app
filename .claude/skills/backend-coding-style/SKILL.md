# backend-coding-style (코드 스타일 규약)

**JavaScript만 사용 — TypeScript 절대 금지** (`.ts` 파일 생성 금지).

## 주석 규칙

- 주석은 이해하기 어려운 코드에만 최소한으로 단다. 코드만 읽어도 알 수 있는 내용에는 달지 않는다.
- 왜 그렇게 했는지(의도·제약)가 코드로 안 드러날 때만 짧게 단다. 예: 왜 특정 상태에서만 no-op으로 처리하는지, 왜 특정 순서로 호출해야 하는지.

## 모듈 분리

- 역할별로 파일을 나눈다: 창 관리(`windows/`) / IPC 등록(`ipc.js`) / 상태(`store/`) / 저장(`data/`). 새 코드를 기존 파일에 몰아넣지 않는다.
- 기존 코드의 구조 변경(리팩토링)은 사용자가 명시적으로 요청할 때만 한다.

## 일반 규칙

- 명시되지 않은 npm 패키지 임의 설치 금지. 필요하면 먼저 확인받는다.
- 구현 후 테스트 실행 금지. 테스트 작성·실행·빌드 검증은 사용자가 명시적으로 요청할 때만.
- 에러 발생 시 임의로 우회하지 말고 상황을 보고하고 방향을 확인받는다.
