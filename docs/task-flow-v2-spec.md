# 태스크 흐름 v2 — "묻지 않는 자동 흐름" 명세

> 작성일: 2026-07-15
> 관련 문서: [floating-widget-spec.md](floating-widget-spec.md), [user-flow-home.md](user-flow-home.md)
> **이 문서는 floating-widget-spec.md의 "다음 활동 팝업(이어서 하기/다음으로/휴식/멈추기 분기)" 설계를 대체한다.**

---

## 1. 배경 / 문제

기존 설계는 테스크가 끝나는 순간마다 사용자에게 선택지를 물었다(휴식하기 / 다음으로 / 이어가기).
그 결과:

- 위젯·팝업에 버튼과 탭이 계속 늘어남 ("다음으로" 아이콘까지 추가하면 위젯이 과밀해짐)
- 매 테스크마다 휴식 여부를 일일이 선택해야 해서 피로함
- 완료된 할일 목록이 위젯에 함께 떠서 메인 화면 주변이 산만함

## 2. 설계 원칙

**테스크 종료 시 사용자에게 묻지 않는다. 설정된 기본 흐름대로 자동 진행하고, 흐름 안에 "건너뛰기"만 둔다.**

- 선택은 매번이 아니라 **설정에서 한 번**만 한다.
- 위젯은 "지금 뭘 하고 있고 얼마나 남았나" 하나만 답한다. 완료 목록·다음 목록은 위젯의 역할이 아니다.

## 3. 위젯 컨트롤 (확장 상태 기준)

버튼은 **2개만** 둔다. "다음으로" 아이콘은 만들지 않는다.

| 버튼 | 의미 | 동작 |
|------|------|------|
| ⏸ 일시정지 | 타이머만 멈춤 | 다시 누르면 재개 (토글 ⏸/▶) |
| ■ 끝내기 | 이 테스크 완료 | 완료 기록 → 자동 흐름 진입 (아래 상태 머신) |

- 미니 버블 ↔ 확장, hover X(위젯 숨김), 종료 5분 전 재등장 규칙은 floating-widget-spec.md를 그대로 따른다.
- 위젯에서 **완료된 할일 목록(WidgetTaskList의 done 섹션)은 제거**한다. 대신 진행률 뱃지 하나만 표시: `3/7` (완료/전체).

## 4. 상태 머신

```
[진행중]
   │ ⏸ ──▶ [일시정지] ── ▶ ──▶ [진행중]
   │
   │ ■ 클릭  또는  타이머 만료
   ▼
(테스크 완료 기록: 예정 시간 vs 실제 소요 시간)
   │
   ├─ 남은 테스크 없음 ──────────▶ [DaySummary 팝업]  ── 확인 ──▶ 세션 종료
   │
   ├─ 휴식 설정 = 없음 ──────────▶ [다음 테스크 대기]
   │
   └─ 휴식 설정 = N분 ──▶ [휴식 화면 (N분 카운트다운)]
                              │ 카운트다운 종료 (자동)
                              │ 또는 "건너뛰기" 클릭
                              ▼
                        [다음 테스크 대기]
                              │ "시작" 클릭
                              ▼
                          [진행중]
```

- **타이머 만료와 ■ 클릭은 같은 경로**를 탄다. 별도 알람 팝업 분기 없음.
- 기존의 "이어서 하기(N분 연장)"는 흐름에서 제거한다. 더 하고 싶으면 [다음 테스크 대기]에서 시작을 누르지 않고 있으면 된다. (연장 니즈가 실사용에서 확인되면 대기 화면에 "+10분 이어하기" 링크로 추후 추가 — v2 범위 외)

### 4.1 휴식 화면

- 큰 카운트다운 + **"건너뛰기"** 링크 하나. 일시정지·연장 컨트롤은 두지 않는다 (단순 유지).
- 휴식 화면이 곧 "휴식 탭"을 대체한다. **메인창의 휴식하기/다음으로/이어가기 탭은 삭제.**

### 4.2 다음 테스크 대기 화면 (NextTaskOverlay 재사용)

- 다음 테스크 이름 + 예정 시간 + **[시작]** 버튼.
- **"다른 테스크 선택"** 링크: 목록을 펼쳐 순서와 무관하게 선택 가능 (건너뛴 테스크는 미완료로 남음).
- 우상단 X: 오버레이 숨김(결정 보류, 세션 유지) — 기존 규칙 유지.

### 4.3 DaySummary 팝업

- 트리거: 마지막 테스크 완료 시 자동. 추가로 [다음 테스크 대기]에 **"오늘 그만하기"** 링크를 두어 수동 트리거 가능 (미완료 테스크는 미완료로 기록).
- 내용: 오늘 한 일 목록, 테스크별 **예정 vs 실제 소요 시간**, 총 집중 시간, 총 휴식 시간.

## 4.4 창별 표시 규칙

상태 머신은 메인 프로세스(sessionStore)가 단일 소스이며, **메인창과 위젯 모두 같은 상태를 반영**한다.

| 상태 | 위젯 창 | 메인창 |
|------|---------|--------|
| 진행중 / 일시정지 | 타이머 (버블/알약) | 홈: 현재 테스크 진행 표시 |
| 휴식 | 휴식 카운트다운 화면 | 홈 그대로 (탭 없음, 상태 뱃지 정도만) |
| 다음 테스크 대기 | **대기 화면 (시작/다른 테스크/그만하기)** | 홈: 다음 테스크 하이라이트 |
| DaySummary | 요약 팝업 | 홈: 오늘 한 일 표에 완료 반영 |

- 현재 구현은 NextTaskOverlay가 **메인창에만** 렌더링됨(App.jsx) → 메인창을 숨겨두면 대기 화면을 볼 수 없다. **대기 화면을 위젯 쪽으로 이동(또는 위젯에도 표시)**하는 것이 v2의 변경 사항. 자동 흐름의 모든 단계(휴식 → 대기 → 요약)는 위젯 창 하나로 완결되어야 한다.
- 메인창은 조회·편집(할일 관리, 기록, 설정) 담당, 위젯은 실행 흐름 담당으로 역할을 분리한다.

## 5. 설정 추가

`settings.json`에 1개 항목만 추가:

| 키 | 값 | 기본값 |
|----|-----|--------|
| `restBetweenTasks` | `0`(없음) / `5` / `10` / 커스텀 분 | `5` |

"매번 물어보기" 옵션은 두지 않는다 — 묻지 않는 것이 이 설계의 핵심이므로.

## 6. 숨김 상태에서의 재등장 (검증 항목)

메인창·위젯이 **둘 다 숨겨진 상태**에서도 다음 시점에 위젯(또는 해당 화면)이 반드시 자동 표시되어야 한다:

1. 타이머 종료 5분 전 (기존 규칙)
2. 타이머 만료 → 휴식 화면 또는 다음 테스크 대기
3. 휴식 카운트다운 종료 → 다음 테스크 대기
4. 마지막 테스크 완료 → DaySummary

체크: `show()` + `alwaysOnTop` 동작, macOS에서 다른 앱 전체화면일 때 표시 여부.

## 7. 버그 수정 (v2에 포함)

- **메인창 스크롤 불가**: 콘텐츠가 창 높이를 넘어도 스크롤되지 않음. 레이아웃 컨테이너의 `overflow`/높이 설정 점검 필요.

## 8. 삭제 목록

- 메인창: 휴식하기 / 다음으로 / 이어가기 탭
- 위젯: 완료된 할일 목록 표시
- 타이머 만료 시 선택지 팝업(AlarmPopup의 휴식/계속 분기)
- 흐름상의 "이어서 하기(연장)" 버튼

## 9. 보류 / 추후 결정

- 일시정지 중 앱 종료 시 세션 복원 정책 (현행 sessionStore 영속화 동작 확인 후 결정)
- 대기 화면 "+N분 이어하기" 링크 (실사용 피드백 후)
- 휴식 자동 시작 대신 3초 유예("곧 휴식이 시작됩니다") 필요 여부
