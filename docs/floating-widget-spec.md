## 1. 원하는 동작 (요구사항)

| # | 요구 동작 |
|---|-----------|
| 1 | **시작하기** 시 플로팅 버튼은 **기본 상태(미니 버블)**로 화면 구석에 뜬다. 미니 버블에 **마우스를 올리면 오른쪽 상단에 X 버튼**이 나타나고, X를 누르면 **플로팅 버튼만 사라진다** (타이머는 계속 진행). |
| 2 | 미니 버블을 **클릭하면 확장 상태**로 전환. 확장 상태의 버튼은 일시정지·X가 아니라 **멈춤(■) 버튼 하나**다. 멈춤 = "이 활동 그만". |
| 3 | 멈춤 클릭 시 해당 활동을 **종료**하고, **"다음 활동 할건지" 팝업**을 띄운다. |
| 4 | X로 숨긴 플로팅 버튼은 **타이머 종료 5분 전에 다시 나타난다**. |
| 5 | 타이머가 끝나면 **다음 활동 할건지 물어보는 팝업**을 띄운다. (user-flow-home.md의 3분기: 이어서 하기 / 다음으로 / 휴식) |
| 6 | 확장 상태에 **축소(미니로) 버튼**이 있다. 미니↔확장은 사용자가 직접 선택하며, 선택한 보기 상태가 유지된다. |
| 7 | 타이머 종료 팝업(휴식 물어보는 팝업)에 다음이 있어야 한다: **우상단에 X 버튼을 표시**하고, 누르면 **팝업이 화면에서 사라진다**(앱이 안 보이게). 결정은 보류되고 세션 상태는 유지된다. 그 외 **멈춤**(이 활동 완료 종료), **다음으로 넘어가기**(다음 할 일 타이머 즉시 시작) 버튼. |

### 상태 전이 요약

```
시작하기 ──▶ [미니 버블 (기본)]
                │  hover: 우상단 X 표시
                │    X 클릭 ──▶ 위젯 숨김 (타이머 유지) ──▶ 종료 5분 전 자동 재등장
                │  클릭
                ▼
          [확장 알약]  ── 축소 버튼 클릭 ──▶ 미니로 복귀 (사용자 선택, 상태 유지)
                │  멈춤(■) 클릭
                ▼
          활동 종료 + [다음 활동 팝업]

타이머 종료 ──▶ [다음 활동 팝업]
                ├─ 우상단 X: 팝업·위젯 숨김 (결정 보류, 세션 유지 — 돌아오면 다시 보임)
                ├─ 이어서 하기: N분 연장 (시간 누적 합산)
                ├─ 다음으로 넘어가기: 다음 할 일 타이머 즉시 시작
                ├─ 휴식: 휴식 시간 설정으로
                └─ 여기서 멈추기: 활동 완료 종료 → 다음 할 일 "시작하기" 대기
```

## 2. 현재 구현과의 차이 (Gap 분석)

| # | 요구 | 현재 상태 | 판정 |
|---|------|-----------|------|
| 1a | 시작 시 미니 버블 | 항상 **확장 알약만** 렌더링 (`TimerWidget.jsx`). 미니 상태 자체가 없음 | ❌ 미구현 |
| 1b | hover 시 우상단 X → 위젯만 숨김 | 위젯에 숨김 X 없음. 이전의 창 닫기(X)는 프레임리스 전환으로 UI가 사라졌고, `widgetWindow.js`의 `close` 핸들러(xButtonMode 분기)만 잔존 | ❌ 미구현 |
| 2 | 클릭 → 확장, 버튼은 멈춤(■) 하나 | 확장 알약에 **일시정지(⏸) + X** 두 개. X가 `endFocus()` 호출 | ❌ 다름 |
| 3 | 멈춤 → 활동 종료 + 다음 활동 팝업 | `endFocus()`는 **휴식 설정 화면(restSetup)**으로 이동 — 수동 멈춤인데 휴식 멘트가 나옴 (user-flow-home.md #1 위반) | ❌ 다름 |
| 4 | 종료 5분 전 위젯 재등장 | 해당 로직 없음. `_tick()`은 알람만 발화 (`sessionStore.js:358`) | ❌ 미구현 |
| 5 | 종료 시 다음 활동 팝업 | `AlarmPopup`이 뜨긴 하나 선택지가 **휴식/계속하기 2개**뿐 — "다음으로 넘어가기" 없음 | ⚠️ 부분 구현 |

## 3. 수정 계획

### 3-A. 디자인 (timestamp-home.pen)

| 항목 | 수정 내용 |
|------|-----------|
| `c9s8D` 확장 알약의 버튼 | Pause(⏸) + X 두 개 → **멈춤(■, lucide `square`) 하나**로 교체 (요구 #2) |
| `s0O8rS` 미니 버블 | hover 상태 변형 추가: **우상단에 작은 X 배지**(예: 18×18, 흰 배경 + 그림자, 버블 우상단 겹침) |
| `c9s8D` 확장 알약의 축소 버튼 | 멈춤(■) 옆에 **축소(미니로) 버튼** 추가 (lucide `minimize-2` 또는 `chevrons-down-up`) — 요구 #6 |
| 다음 활동 팝업 | "○○ 끝났어요 — 다음 활동 할까요?" 팝업 프레임 신규 (기존 Timer Complete Popup `cTcXG` 확장 여부 확인). 구성: **우상단 X**(숨김) + 주 버튼 3개 **이어서 하기 / 다음으로 넘어가기 / 휴식** + 보조 텍스트 버튼 **"여기서 멈추기"**(활동 완료 종료) — 요구 #5, #7 |

### 3-B. 백엔드 (main 프로세스)

| 파일 | 수정 내용 |
|------|-----------|
| `windows/widgetWindow.js` | 크기 상태 추가: `mini`(64+여백 ≈ 88×88), `pill`(324×84), `panel`(324×204). 렌더러가 미니↔확장을 전환할 수 있도록 `widget:setMode` IPC로 크기 변경 함수 노출. 잔존한 `close` 이벤트의 xButtonMode 분기는 "위젯 숨김 ≠ 세션 취소" 원칙에 맞게 정리 |
| `store/sessionStore.js` | ① `stopTask()`(가칭) 신설: 활동 종료(시간 기록·완료 표시) 후 **restSetup이 아니라** 다음 할 일 "시작하기" 대기 상태로 전이 — 요구 #3, #7의 멈춤. ② `_tick()`에 **남은 시간 300초 도달 시 `widget:reappear` 이벤트** emit — 요구 #4. ③ `skipToNext()`(가칭) 신설: 알람 상태에서 현재 활동 기록 후 **다음 할 일 타이머 즉시 시작**(내부적으로 `confirmNextTask()` 재사용 — 마지막 할 일이면 dayFinished) — 요구 #5, #7의 "다음으로". ④ 알람 팝업의 X는 세션을 바꾸지 않는다 — `alarm` 상태 유지 채로 위젯만 숨김. 트레이/메인 창에서 위젯을 다시 열면 팝업이 그대로 보인다 |
| `index.js` | `widget:reappear` 수신 시 `showWidgetWindow()` 호출 (숨김 상태였어도 재등장) |
| `ipc.js` + `preload/index.js` | `widget:hide`(X로 숨김), `widget:setMode`(mini/expanded), `session:stopTask`, `session:skipToNext` 채널 추가 — backend-ipc-conventions 규약대로 `ipcMain.handle` + preload 노출 |

## 4. 정책 결정 사항 (확정)

> 공통 원칙: **숨김/축소는 사용자의 "조용히 해줘" 신호로 존중하고, 파괴적인 동작(활동 종료)만 명시적 버튼으로 남긴다.** (user-flow-home.md의 "간섭하지 않기" 원칙과 일관)

| 항목 | 결정 | 이유 |
|------|------|------|
| 확장 → 미니 복귀 | **확장 상태에 명시적 축소(미니로) 버튼 추가 — 사용자가 미니/확장을 직접 선택하고, 선택한 상태가 유지된다** | 자동 축소는 사용자가 "계속 보고 싶은" 경우와 충돌. 보기 상태는 사용자의 선택으로 존중하고, 앱이 임의로 바꾸지 않는다 |
| 일시정지 | **위젯에서 제거, 홈 화면에만 유지** | "확장 = 멈춤 버튼 하나"의 단순함 유지. 일시정지는 저빈도 예외 상황이라 메인 창을 여는 비용 허용 |
| 5분 전 재등장 상태 | **미니로 등장** | X로 숨긴 것은 "방해하지 마" 의사표시 — 재등장도 최소 존재감으로. 급함 신호는 마지막 5분간 링 색 강조/펄스로 표현 |
| `xButtonMode` 설정 | **제거** | hover X = 숨김(세션 유지), 멈춤 ■ = 활동 종료로 역할이 명시적으로 분리되어 설정으로 고를 것이 없음. `widgetWindow.js`의 close 핸들러는 "무조건 숨김"으로 단순화. 설정 화면에서 해당 옵션 UI도 제거 |

## 5. 구현 순서 제안

1. 백엔드: `stopTask()` + `nextTaskPrompt` 상태 + 5분 전 이벤트 (`sessionStore.js`)
2. IPC/창: `widget:hide`·`widget:setMode` 채널 + 창 크기 3단계 (`widgetWindow.js`, `ipc.js`, preload)
3. 프론트: MiniBubble/ExpandedPill 분리 + hover X + 멈춤 버튼 (`TimerWidget.jsx`)
4. 팝업: AlarmPopup 3분기 확장 + `nextTaskPrompt` 분기 (`WidgetApp.jsx`)
5. 디자인: 펜슬 `c9s8D` 버튼 교체·미니 hover 변형 추가 후 화면과 대조 검증
