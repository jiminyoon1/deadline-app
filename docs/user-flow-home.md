# 홈 화면 유저 플로우 변경 명세 (코드 대조 완료)

> 작성일: 2026-07-14
> 목적: 목표 유저 플로우와 **현재 코드의 실제 동작**을 항목별로 대조하고, 무엇을 어디서 고쳐야 하는지 명시한다.
> 대조 기준 코드: `src/main/store/sessionStore.js`, `src/renderer/src/components/home/TimerPanel.jsx`, `TaskInput.jsx`, `src/renderer/src/components/timer/AlarmPopup.jsx`, `TimerWidget.jsx`, `src/renderer/src/App.jsx`

## 1. 핵심 컨셉 (변경 없음)

**"오늘 할 일을 순서대로 정리해두면, 앱이 그 순서대로 자연스럽게 이어서 진행시켜준다."**

- 사용자는 하루를 시작할 때 오늘 할 일을 **진행할 순서대로** 등록한다.
- 진행 순서는 **나열된 순서가 기본** (별도 선택이 없으면 위에서 아래로).
- 타이머가 끝날 때마다 앱이 "다음 행동"을 묻고, 선택에 따라 흐름이 끊기지 않고 이어진다.

## 2. 목표 유저 플로우

```
[할 일 등록 (순서대로)]
        │
        ▼
[첫 번째 할 일 시작] ──── 타이머 진행 ────▶ [타이머 종료 알람]
                                              │
              ┌───────────────┬───────────────┤
              ▼               ▼               ▼
        ① 이어서 하기    ② 다음으로 넘어가기   ③ 휴식하기
```

- **① 이어서 하기**: 팝업이 닫히고 **바로 위젯으로 복귀**, 분 입력 없음. 초과분은 `+7:32`처럼 **카운트업으로 표시**되고 최종 기록에 누적 합산된다.
- **⓪ 무응답 기본값**: 알람에 아무 응답이 없으면 **"이어서 하기"와 동일하게 동작** — 현재 할 일이 계속 진행되며 +시간이 쌓인다. (자리를 비워도 시간이 붕 뜨지 않음)
- **② 다음으로 넘어가기**: 선택 즉시 나열 순서상 다음 할 일 타이머가 **확인 화면 없이** 바로 시작.
- **③ 휴식하기**:
  - (a) 시간 지정(예: 30분) → 종료 시 "다음 활동 시작하기" 팝업 → "시작" 누르면 다음 할 일 시작.
  - (b) "나중에 다시 오기" → 세션 종료. **사용자가 돌아올 때까지 어떤 팝업/알림도 띄우지 않는다.**

## 3. 항목별 변경 명세 (현재 동작 → 목표 동작)

### 3-1. 알람 팝업: 2개 선택지 → 3개 선택지 + 무응답 기본값

- **현재**: 메인 창의 `AlarmPopup.jsx`는 **"계속하기"(overtime 진입)와 "휴식"(endFocus) 2개**만 제공한다. 스토어에는 `continueFromAlarm()`(overtime 카운트업), `skipToNext()`, `stopTask()`가 구현돼 있다.
- **목표**:
  1. 알람 팝업에서 **① 이어서 하기(`continueFromAlarm` — 분 입력 없이 즉시 위젯 복귀, 초과분 카운트업) ② 다음으로 넘어가기(`skipToNext`) ③ 휴식하기(`endFocus` → restSetup)** 3개를 제시한다.
  2. **무응답 시 기본값 = 이어서 하기**: 팝업에 응답이 없으면 현재 할 일이 계속 진행되며 +시간이 쌓인다. (팝업이 떠 있는 동안에도 시간 계측이 멈추지 않도록 — 응답 시점에 소급 반영하거나, 알람 상태에서도 tick 유지)
  3. 초과 시간은 위젯/타이머에 `+MM:SS` 형태로 표시 (현재 위젯의 `+플로우 집중 중` 표시와 동일 계열).
- **수정 위치**: `src/renderer/src/components/timer/AlarmPopup.jsx`, 이를 렌더하는 `TimerWidget.jsx` 및 메인 창 알람 표시 경로. `sessionStore.js`의 alarm 상태 tick 처리 확인.
- **참고**: 분 입력 방식 연장(`extendFocus`)은 이 결정으로 불필요해짐 — 다른 곳에서 안 쓰면 관련 IPC와 함께 정리.

### 3-2. 멈춤(■) 버튼 = 완료 처리 (휴식 제안 없음)

- **현재**: `TimerPanel.jsx`의 ■ 버튼은 `endFocus()`를 호출한다. `endFocus`는 태스크를 **완료 처리하지 않고** `restSetup`(휴식 제안 화면)으로 보낸다. 즉 "직접 멈췄는데 휴식을 물어보는" 문제가 여기서 발생한다.
- **목표**:
  1. ■ = "이 할 일은 이제 그만" → 태스크 **완료 처리**(기존 `stopTask()`가 이 동작: `_finishActiveTask()` 후 idle). **휴식 제안 팝업을 띄우지 않는다.** 토스트 등 확인 UI도 없이 조용히 처리한다.
  2. 완료 즉시 **다음 순서 할 일의 "시작하기" 대기 화면**(preview)으로 자동 전환.
  3. 오조작 구제는 토스트가 아니라 **"다시 시작"으로 해결**: 완료된 할 일 행에 **다시 시작 버튼**을 두어, 누르면 완료가 해제되고 해당 태스크의 타이머가 이어서 시작된다. **이미 집중한 시간은 그대로 유지·누적**된다. 완료 행의 시간 표기는 `25분 계획 · 18분 집중` 형식.
- **수정 위치**:
  - `TimerPanel.jsx`: ■ 버튼 핸들러를 `endFocus` → `stopTask`로 교체.
  - `sessionStore.js`: **다시 시작 액션 신규 구현** — `restartTask(id)`: `completed = false`, `status` 복원 후 `startTask` 흐름으로 진입 (진행 중 타이머가 있으면 §3-4 규칙대로 정리). `startTask`가 완료 태스크를 거르지 않는지 확인.
  - IPC: `restartTask` 채널 추가 (`src/main/ipc.js`, `src/preload/index.js` — backend-ipc-conventions 스킬 준수).
  - `TaskList.jsx`/`TaskRow.jsx`: 완료된 행에 다시 시작 버튼 + `계획 · 집중` 시간 표기.
  - `App.jsx`: stopTask 후 다음 pending 태스크를 `previewTaskId`로 설정.

### 3-3. 일시정지(⏸) ↔ 재개(▶)

- **현재**: `TimerPanel.jsx`가 이미 paused일 때 ▶로 바뀌고 `resumeTask()`를 호출한다. 스토어의 `pauseTask`/`resumeTask`도 완료 처리와 무관하다. **✅ 이미 목표대로 동작 — 변경 없음.** (3-2 적용 후에도 ■과 ⏸의 의미 구분이 유지되는지만 확인)

### 3-4. 시작하기 = 단일 실행

- **현재**: `sessionStore.startTask()`는 `timer.status`가 `idle`/`restFinished`가 아니면 **아무것도 하지 않는다**. 즉 다른 타이머가 돌고 있으면 시작이 "무시"된다 (다른 타이머를 끄고 시작하는 게 아님).
- **목표**: "시작하기"를 누르면 **기존 진행 중 타이머를 정리(해당 태스크는 paused/pending으로 되돌리고 경과시간 보존)한 뒤** 선택한 태스크의 타이머만 시작한다. 동시에 두 타이머가 도는 상태는 어떤 경로로도 불가능해야 한다.
- **수정 위치**: `sessionStore.js`의 `startTask()` 가드 로직 변경.

### 3-5. 진행 순서 = 나열 순서

- **현재**: `confirmNextTask()`가 현재 태스크 인덱스 기준으로 **순환하며 다음 미완료 태스크**를 찾는다. `skipToNext()`와 휴식 후 시작 모두 이를 사용. **✅ 대체로 목표와 일치.**
- **확인 필요**: 리스트 끝에서 순환(wrap-around)하는 것이 의도인지 — 마지막 할 일 이후 처리(§5 열린 논의)와 함께 결정.

### 3-6. 입력 폼: 분 → 이름으로 ← 키 복귀

- **현재**: `TaskInput.jsx`는 이름 입력칸에서 →키로 분 칸으로 이동하는 것만 구현(`handleNameKeyDown`). **분 칸에서 ←키로 돌아오는 핸들러가 없다.**
- **목표**: 분 입력칸에서 커서가 맨 앞일 때 ←를 누르면 이름 입력칸으로 포커스 복귀.
- **수정 위치**: `TaskInput.jsx` — minutes input에 `onKeyDown` ArrowLeft 처리 추가 (`selectionStart === 0`일 때 `nameRef.current?.focus()`).

### 3-7. 타이머 아래 시간 추가(+) 버튼 제거 (결정됨)

- **현재**: `TimerPanel.jsx`에 + 버튼(10분 고정, `ADD_FOCUS_SECONDS`)이 있고 스토어의 `addFocusTime()`을 호출한다.
- **목표**: **+ 버튼 제거.** 연장은 알람 시점의 "이어서 하기"(`extendFocus`)로만 가능. (알람 전 선제 연장은 데드라인 압박을 무디게 함)
- **수정 위치**: `TimerPanel.jsx`에서 버튼·`PlusIcon`·`onAddFocusTime` prop 제거, `App.jsx`에서 전달 제거. 다른 곳에서 안 쓰면 `sessionStore.addFocusTime()`과 관련 IPC 채널도 함께 정리.

### 3-8. 휴식 (b) "나중에 다시 오기" — 무간섭

- **현재**: `cancelSession()`/`backToHome()`이 타이머를 idle로 되돌린다. resting 종료 시에만 `restFinished` 팝업이 뜨므로 큰 문제는 없어 보이나, "나중에 다시 오기" 경로가 UI상 명확한지 확인 필요.
- **목표**: 휴식 설정 화면에서 "나중에 다시 오기" 선택 시 세션을 idle로 종료하고 **이후 어떤 팝업/알림도 없다.** 복귀 시 홈에 **마지막 진행 태스크가 보이도록** (previewTask로 표시).
- **수정 위치**: 휴식 설정 UI(`RestScreen.jsx` 등)와 `App.jsx`의 복귀 컨텍스트 표시.

### 3-9. 플로팅 위젯: X = 위젯 숨기기, 완료(■) 버튼 신설

- **현재**: `TimerWidget.jsx`의 X 버튼이 `endFocus()`(휴식 제안 진입)를 호출한다. 위젯을 치우고 싶을 뿐인데 세션 흐름이 바뀐다.
- **목표**:
  1. **X 버튼 = 위젯 창 숨기기.** 세션·타이머는 그대로 계속 진행된다. (창 숨김은 IPC로 메인 프로세스에 요청 — backend-window-management 규약)
  2. **■(완료) 버튼을 별도로 추가** — 홈 화면 ■과 동일하게 `stopTask()` 호출, 완료 처리 + 되돌리기 토스트(§3-2)와 같은 의미.
  3. 위젯 버튼 구성: ⏸/▶ (일시정지/재개) · ■ (완료) · X (위젯 숨기기).
- **수정 위치**: `TimerWidget.jsx`(버튼 구성), `src/main/windows/widgetWindow.js` + IPC(위젯 숨기기 채널).

### 3-10. 자정 롤오버: 사용 중이면 세션을 날리지 않는다

- **현재**: `sessionStore.js`의 `_checkMidnightRollover()`가 날짜가 바뀌는 순간 **타이머를 무조건 초기화**한다. 밤 11시 50분에 시작한 타이머가 자정에 조용히 사라진다.
- **목표**:
  1. **자정에 진행 중(running/paused/alarm/resting)인 세션은 그대로 유지**한다. 사용자가 쓰고 있는 동안에는 날짜가 바뀌어도 아무 일도 일어나지 않는다.
  2. 진행 중이던 태스크의 기록은 **시작한 날짜에 귀속**한다 (자정을 걸쳐도 기록이 쪼개지거나 사라지지 않음).
  3. 날짜 전환(새 record 생성)은 **세션이 idle로 돌아온 시점**에 수행한다.
- **수정 위치**: `sessionStore.js` `_checkMidnightRollover()` — 타이머가 idle이 아닐 때는 롤오버를 보류.

## 4. 용어 정리 (혼동 방지)

| 용어 | 의미 | 스토어 액션 |
|------|------|------------|
| **멈춤 (■)** | 이 할 일을 끝내겠다. 완료 처리 후 다음 일정 대기 (팝업·토스트 없음) | `stopTask()` |
| **다시 시작** | 완료된 할 일을 다시 진행. 완료 해제 + 타이머 재개, 기존 집중 시간 유지·누적 | 신규 `restartTask(id)` |
| **일시정지 (⏸)** | 잠깐 시간만 멈춤. 완료 아님. ▶로 재개 | `pauseTask()` / `resumeTask()` |
| **이어서 하기** | 같은 할 일 계속. 분 입력 없이 초과분 카운트업(+MM:SS), 누적 합산. 알람 무응답 시 기본값 | `continueFromAlarm()` |
| **다음으로 넘어가기** | 현재 확정 후 다음 순서 할 일 즉시 시작 | `skipToNext()` |
| **X (위젯)** | 위젯 창만 숨김. 세션·타이머는 계속 진행 | 신규 IPC (창 숨기기) |
| **휴식하기** | 휴식 시간 설정 화면으로 | `endFocus()` → `startRest()` |
| **나중에 다시 오기** | 세션 종료. 복귀 전까지 무간섭 | `cancelSession()` |

## 5. 확정된 UX 원칙

- **알람 종료 vs 수동 멈춤 구분**: 알람이 울리면 앱이 묻는다(이어서/다음/휴식). 사용자가 직접 ■를 누르면 이미 결정한 것 — 묻지 않고 완료 처리 후 다음 일정으로.
- **집중한 시간은 어떤 경우에도 사라지지 않는다**: 강제 태스크 전환(§3-4), 완료 후 다시 시작(§3-2), 자정 롤오버(§3-10), 앱 재시작 — 모든 경로에서 이미 계측된 집중 시간은 보존·누적된다. 새 기능을 설계할 때 이 원칙을 기준으로 판단한다.
- **완료 오기록 구제는 "다시 시작"으로**: 실행취소 토스트는 두지 않는다. 완료된 할 일은 언제든 다시 시작할 수 있으므로 별도 undo가 필요 없다.
- **시간 표시 형식**: `25분 계획 · 18분 집중`. 계획 초과 시 `+7분` 강조 가능.
- **복귀 시 컨텍스트 표시**: 나갔다 돌아오면 홈에 마지막 진행 태스크가 보이도록.

## 7. 구현 완료 내역 (2026-07-15)

코드가 문서 작성 시점 이후 발전하여(미니/확장 위젯, NextTaskPopup 3분기 등) 이미 구현된 항목은 생략하고, 이번에 추가 구현한 것만 기록한다.

- **알람 시 위젯 자동 표시**: 위젯을 숨겨놨어도 타이머가 끝나면 위젯이 다시 나타나 다음 행동을 묻는다. (`src/main/index.js` alarm 핸들러)
- **"다음으로" 후 확장 위젯**: 알람/다음 행동 팝업에서 다음 태스크로 넘어가면 위젯이 미니가 아닌 **확장(알약) 상태**로 나와 지금 뭘 하는지 보여준다. 홈에서 새로 시작한 경우는 기존대로 미니. (`TimerWidget.jsx`)
- **확장 위젯 아이콘 정리**: ⏸/▶(일시정지·재개)와 ■(멈춤 → 다음 행동 팝업) 두 개만. 접기는 알약 본문 클릭. (`TimerWidget.jsx`)
- **메인 창 플로우 집중 표시**: 초과(overtime) 중이면 원형 타이머가 가득 찬 링 + `+MM:SS / 플로우 집중 중`으로 표시. (`TimerPanel.jsx`, `CircularTimer.jsx`)
- **메인 창 ■ = 선택 팝업**: 멈춤을 누르면 휴식 화면으로 직행하지 않고 메인 창 안에 **이어서/다음으로/휴식/여기서 멈추기** 팝업이 뜬다. (`NextTaskOverlay.jsx` 신규, `App.jsx`, `TimerPanel.jsx`)
- **진행 중 할 일 시간 수정 반영**: 진행/일시정지 중인 태스크의 계획 시간을 수정하면 돌고 있는 타이머의 남은 시간에 즉시 반영된다. (`sessionStore.updateTask`)
- **휴식 화면 X 버튼**: 휴식 설정/휴식 중/휴식 끝 모든 화면에 위젯 숨기기 X 추가. 숨겨도 휴식 타이머는 계속 돈다. (`RestScreen.jsx`)
- **휴식 건너뛰기**: 휴식 설정·휴식 중 화면에 "건너뛰고 다음 할 일 시작" 추가. 지금까지 쉰 시간만 기록하고 다음 태스크를 바로 시작한다. (`sessionStore.skipRest` 신규 + IPC `session:skipRest`)

### 2026-07-15 (2차) — 문서 §3 잔여 항목

- **§3-7 + 버튼 제거**: 타이머 컨트롤에서 시간 추가(+) 버튼 삭제. `addFocusTime` 스토어 액션·IPC 채널도 함께 정리. 연장은 알람 팝업의 "이어서 하기"로 일원화.
- **§3-6 입력 폼 ← 키**: 분 입력칸에서 커서가 맨 앞일 때 ←로 이름 입력칸 복귀. (`TaskInput.jsx`)
- **§3-4 시작하기 = 단일 실행**: 다른 태스크의 타이머가 돌고 있어도 "시작하기"가 무시되지 않고, 기존 태스크를 대기로 되돌린 뒤(경과시간 보존) 선택한 태스크만 시작. (`sessionStore.startTask`)
- **§3-2 다시 시작 버튼**: 완료된 할 일 행에 ▶ 버튼 — 완료 해제 후 타이머를 이어서 시작, 기존 시간 유지·누적. (`restartTask` 스토어+IPC, `TaskRow.jsx`) ※ "조용히 완료+토스트" 부분은 이후 결정(■=선택 팝업)으로 대체되어 구현하지 않음.
